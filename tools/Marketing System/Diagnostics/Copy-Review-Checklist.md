# Copy Review Checklist


**Purpose:** Systematic checklist for reviewing marketing copy before publishing.

## Quick Review (5-Minute Check)

### Hook
- [ ] First line earns attention (pattern interrupt, curiosity, or call-out)
- [ ] Speaks to Prime Prospect specifically (not everyone)
- [ ] Clear benefit or intrigue within first 10 seconds of reading

### Promise
- [ ] Clear transformation or outcome stated
- [ ] Specific and believable (not vague or hyperbolic)
- [ ] Addresses a real desire or pain

### Proof
- [ ] Evidence supports the promise (testimonials, data, demonstration)
- [ ] Proof is relevant and believable
- [ ] Social proof from relatable sources

### CTA
- [ ] Clear next step
- [ ] Benefit-oriented CTA text (not just "Submit")
- [ ] Single primary CTA (not competing options)

## Full Review by Framework

### Capture-Connect-Convert Check
(From `Core/Core-Principles.md`)

**Capture:**
- [ ] Hook grabs attention in first sentence/frame
- [ ] Pattern interrupt or specificity earns the stop
- [ ] Audience knows this is for them within seconds

**Connect:**
- [ ] Empathy demonstrated (you understand their situation)
- [ ] Story or relatability builds rapport
- [ ] Beliefs begin shifting through insight or revelation

**Convert:**
- [ ] Offer clearly presented
- [ ] Value justifies the ask
- [ ] CTA is clear and compelling

---

### Belief Barriers Check
(From `Psychology/Belief-Mechanics.md`)

**Method Barrier:**
- [ ] The approach/vehicle is explained
- [ ] Why this method works is addressed
- [ ] Differentiation from failed alternatives

**Self Barrier:**
- [ ] "I can do this" is addressed
- [ ] Proof from people like them
- [ ] Simplicity or support emphasized

**Circumstance Barrier:**
- [ ] "Now is the right time" is addressed
- [ ] Urgency or opportunity framed
- [ ] Obstacles acknowledged and handled

---

### Offer Stack Check
(From `Persuasion/Offer-Design.md`)

- [ ] **The Deliverables:** What they get is clear
- [ ] **How They Get It:** Delivery method explained
- [ ] **Why It Works:** Mechanism/approach justified
- [ ] **Why Now:** Urgency or timing addressed
- [ ] **How Long:** Timeframe and expectations set
- [ ] **How Much:** Value and price anchored
- [ ] **Risk Removal:** Guarantee clearly stated
- [ ] **Bonuses:** Additional value stacked (if applicable)
- [ ] **Scarcity:** Real limitation communicated (if applicable)

---

### Trust Signals Check
(From `Psychology/Trust-Architecture.md`)

- [ ] Credibility established (expertise, results, credentials)
- [ ] Social proof included (testimonials, logos, numbers)
- [ ] Authenticity present (real voice, not corporate)
- [ ] Specificity throughout (details, not vague claims)
- [ ] Risk addressed (guarantee, FAQ, objection handling)

---

### Format-Specific Checks

#### Email
- [ ] Subject line earns the open
- [ ] First line earns the next sentence
- [ ] One main idea per email
- [ ] Personal, conversational tone
- [ ] Single CTA (repeated if long)
- [ ] Mobile-friendly formatting (short paragraphs)

#### Sales Page
- [ ] Headline captures core promise
- [ ] Above-fold content earns the scroll
- [ ] Sections flow logically (problem → solution → offer → proof → CTA)
- [ ] FAQ addresses real objections
- [ ] Multiple CTAs throughout
- [ ] Testimonials placed strategically

#### Ad
- [ ] Hook in first 3 seconds (video) or first line (text)
- [ ] Clear benefit or curiosity established
- [ ] CTA matches landing page promise
- [ ] Message-market match with audience targeting
- [ ] Compliant with platform policies

#### Webinar/VSL
- [ ] Pattern interrupt opening
- [ ] Promise and preview in first 2 minutes
- [ ] Three secrets/belief shifts present
- [ ] Transition to offer is smooth
- [ ] Stack builds value before price reveal
- [ ] Close includes urgency and CTA

---

### Ethical Checks

- [ ] Claims are accurate and provable
- [ ] Testimonials are real and representative
- [ ] Scarcity is genuine (not manufactured)
- [ ] No income/results guarantees (unless legally compliant)
- [ ] Disclosed affiliations/sponsorships where required
- [ ] Respects audience intelligence (no manipulation tactics)

---

### Voice and Tone Check

- [ ] Consistent with brand voice
- [ ] Appropriate for audience sophistication
- [ ] Reads naturally (not stiff or over-written)
- [ ] Active voice predominates
- [ ] Jargon appropriate to audience (not excessive)
- [ ] Energy level matches context

---

### Technical Checks

- [ ] Links work correctly
- [ ] Tracking parameters in place (UTMs)
- [ ] Mobile display tested
- [ ] Forms function properly
- [ ] Load time acceptable
- [ ] No typos or grammatical errors

---

## Review Process

### Quick Pass (5 min)
Run through "Quick Review" section.
Flag major issues.

### Deep Pass (15-30 min)
Run through all framework checks.
Note specific improvements.

### Polish Pass (10 min)
Read aloud for flow.
Check technical elements.
Final proof.

---

## Common Copy Failures

| Failure | Symptom | Fix |
|---------|---------|-----|
| **Weak hook** | Low engagement, high drop-off | Stronger pattern interrupt, specificity |
| **Vague promise** | Low interest, "so what?" feeling | Get specific, make it tangible |
| **No proof** | Skepticism, no action | Add testimonials, data, demonstration |
| **Buried CTA** | Confusion, no action | Make CTA clear, repeat, emphasize |
| **Wrong tone** | Disconnect, unsubscribes | Match audience expectations |
| **Too long** | Drop-off before key points | Cut ruthlessly, front-load value |
| **Too short** | Insufficient belief shift | Add proof, story, objection handling |
