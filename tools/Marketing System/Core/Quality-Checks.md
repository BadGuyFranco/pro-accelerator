# Quality Checks

Mandatory quality gates before delivering any marketing output.

## Pre-Delivery Checklist

### 1. Core Principles Verification

**Blueprint:**
- [ ] Prime prospect clearly defined
- [ ] Messaging aligned with prime prospect's desires and language
- [ ] Angle matches motivation (away from pain OR toward pleasure)

**Ascension:**
- [ ] Offer positioned correctly on the ladder
- [ ] Logical next step identified
- [ ] Tier-ups/tier-downs appropriate

**Persuasion Structure:**
- [ ] Clear Hook that interrupts patterns (does NOT sell)
- [ ] Story that creates epiphany or belief shift
- [ ] Offer with clear promise, deliverables, and CTA

**Conviction:**
- [ ] Method barriers addressed
- [ ] Self barriers addressed
- [ ] Circumstance barriers addressed

### 2. Offer Integrity

- [ ] Core promise clear and specific
- [ ] Deliverables explicitly stated
- [ ] Speed/ease elements present
- [ ] Risk reversal included
- [ ] Scarcity or urgency present (if appropriate)
- [ ] Price positioned correctly

### 3. Format-Specific Checks

**Ads:**
- [ ] 3-5 hook variations provided
- [ ] Hook matches traffic warmth
- [ ] Ad pre-frames the landing page
- [ ] Clear CTA

**Sales/Landing Pages:**
- [ ] Strong headline
- [ ] Flow: Hook → Empathy → Story → Pathway → Offer → Stack → Scarcity → Risk Reversal → FAQs
- [ ] Value stack clear
- [ ] Multiple CTAs

**Email Sequences:**
- [ ] Sequence type identified
- [ ] Each email has Hook, Story, Offer
- [ ] Sequence builds toward conversion
- [ ] Each email stands alone

**Webinars/VSLs:**
- [ ] Master conviction statement clear
- [ ] Three secrets/teachings identified
- [ ] Each secret breaks a belief barrier
- [ ] Stack and close structure present

**Funnels:**
- [ ] Each step leads to the next
- [ ] Proper pre-framing between steps
- [ ] Tier-ups/tier-downs positioned
- [ ] Follow-up sequences planned
- [ ] Entry matches traffic temperature

### 4. Multi-Framework Integration

- [ ] Each framework verified individually
- [ ] Foundational frameworks applied throughout
- [ ] No framework conflicts
- [ ] Integration follows: Core > Format-Specific > Supporting

### 5. Measurement Included

- [ ] What to track (leading and lagging indicators)
- [ ] Success criteria defined
- [ ] Test priority identified

### 6. Business Model Alignment

- [ ] Appropriate frameworks loaded
- [ ] No wrong assumptions about business type
- [ ] Platform guidance matches actual channels
- [ ] Growth recommendations align with resources

## Failure Resolution

If any check fails: fix before delivering, re-run check, only deliver when all pass.

## Quick Verification (Simple Outputs)

- [ ] Prime prospect alignment
- [ ] Hook → Story → Offer structure
- [ ] Clear CTA
- [ ] Measurement criteria included
